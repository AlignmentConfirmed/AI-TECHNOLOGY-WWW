# The Flat Web — Why Every Website Lacks Depth and Why It Matters Now

## The Problem

Every website on the internet is a flat rectangle. Top to
bottom. Left to right. One dimension of content rendered
into a two-dimensional visual space. No inside. No outside.
No structural boundary between what is verified and what
is generated. No provenance. No depth.

A peer-reviewed physics paper and a conspiracy blog are
structurally identical in a browser. Same HTML. Same CSS.
Same rendering engine. Same flat rectangle. The reader's
brain does all the work of determining which is
trustworthy. The web provides zero structural signal.

This was tolerable when humans wrote most content and the
volume was manageable. It is not tolerable now. AI is
generating content at scale that goes directly onto flat
pages with no domain classification, no provenance, no
verification status. An AI-generated article is
structurally identical to a human-written article is
structurally identical to a hallucination. All flat. All
rendered the same. All one dimension.

## Why This Is Urgent Now

### Attention spans have collapsed

Human attention spans have shortened to the point where
long text is not processed deeply. Users scan. They skim
headlines. They scroll. They do not read — they sample.

In this environment, depth perception is the first casualty.
The reader cannot determine:
- Is this content verified or generated?
- Is this a primary source or a copy of a copy?
- Who wrote this — a human, an AI, or a human directing AI?
- When was this written and has it been updated?
- Does anyone credible agree with this?

On a flat page, these questions have no structural answer.
The reader must investigate each one manually. With a
shortened attention span, they don't. They accept the flat
page at face value.

### AI content is indistinguishable from human content

AI-generated text on a flat web page looks identical to
human-written text. The formatting is the same. The
confidence is the same. The structure is the same. The
reader has no visual or structural signal that the content
was generated by an AI that may have been in narrative
compensation mode when it produced it.

### Multiple information sources cannot be compared

A person researching a topic has multiple browser tabs open.
Each tab is a flat page. There is no structural connection
between the tabs. The reader must hold the comparison in
their own working memory — which is limited to approximately
7 items. With 12 tabs open, the reader cannot maintain the
dimensional relationships between sources.

Pop-outs, background chats, side panels, and notification
windows compound this. Each is a separate flat rectangle
competing for the same limited attention. The reader
loses track of:
- Which window had the source they were referencing
- Which chat had the conversation they need to continue
- Which calendar event relates to which research thread
- Which git commit corresponds to which document change

The information is all there. The depth is not. The
relationships between information sources are held in the
reader's head, not in the interface. When attention span
is short, those relationships are the first thing lost.

## What Depth Means

Depth in a web interface means structural layers that the
reader can perceive without investigating:

### Layer 1 — Verification depth

How verified is this content?

| Depth | Signal | Current web | What's needed |
|-------|--------|-------------|---------------|
| Unverified | No one has checked this | No signal | Visual indicator |
| Self-reviewed | The author checked their own work | No signal | Visual indicator |
| Peer-reviewed | Another human evaluated it | Sometimes noted in text | Structural metadata |
| Independently verified | Multiple independent sources agree | No signal | Convergence receipt |
| AI-generated | An AI produced this content | Rarely disclosed | Mandatory structural tag |

Currently: all five levels look identical on a flat page.
A reader cannot tell the difference without reading the
content, evaluating the source, and cross-referencing
manually.

### Layer 2 — Provenance depth

Where did this come from?

- Who directed it (human, AI, or collaboration)
- What system generated it (which AI, which version)
- When it was created and last modified
- What domain it was created in (research, creative,
  verified, frontier)
- What its relationship is to other content (cites,
  contradicts, extends, supersedes)

Currently: provenance is either absent or buried in
footer text that nobody reads.

### Layer 3 — Relational depth

How does this content relate to other content the reader
is currently viewing?

- Does this tab confirm or contradict that tab?
- Does this chat conversation relate to this document?
- Does this calendar event connect to this project?
- Does this git commit correspond to this research finding?

Currently: the reader holds all relationships in their
head. Every tab switch resets the context. Every window
switch breaks the thread.

### Layer 4 — Temporal depth

Where is this content in time?

- Is this current or outdated?
- Has this been superseded by newer work?
- How has this content changed since the reader last
  viewed it?
- What was the state of knowledge when this was written?

Currently: timestamps exist but temporal context does not.
A page from 2019 looks identical to a page from 2026.
The reader must check the date manually and assess whether
the content is still valid.

## What the Interface Needs

### Dimensional indicators on every page

Every piece of web content should carry visible structural
indicators for:
- Verification level (unverified → independently verified)
- Provenance (who/what/when/which domain)
- Content type (verified fact, creative work, frontier
  hypothesis, AI-generated)
- Temporal status (current, outdated, superseded)

These indicators should be structural — part of the
rendering, not optional metadata. The reader should
perceive depth without clicking, investigating, or
reading fine print.

### Relational views between open sources

When a reader has multiple information sources open, the
interface should show the relationships between them:
- Agreement and contradiction between sources
- Citation chains (A cites B which cites C)
- Temporal ordering (this was written before that)
- Domain relationships (this is verified, that is frontier)

This is not a search feature. This is a rendering feature.
The relationships are displayed alongside the content, not
discovered through separate investigation.

### Integrated resource access

The tools a person uses while researching should be
accessible from within the information view, not in
separate windows that compete for attention:

- Calendar — what deadlines and events relate to this
  research thread
- Git — what code changes correspond to this document
- Chat — what conversations relate to this topic, with
  context preserved across sessions
- Citations — what published work supports or contradicts
  what the reader is viewing
- Notes — the reader's own annotations on the content,
  persistent across sessions

Currently these are separate applications in separate
windows. The reader switches between them, losing context
at every switch. The integration should be structural —
the resources are accessible from within the content view,
not from a separate application.

### Attention-aware presentation

The interface should adapt to the reader's attention
state:
- Summary view for scanning (headlines, verification
  status, key findings)
- Detail view for reading (full content with depth
  indicators)
- Comparison view for evaluation (multiple sources
  side by side with relationships visible)
- Focus view for deep work (single source, all
  distractions removed, related resources accessible
  but not visible until requested)

The reader should not have to manage their own attention
through tab management and window arrangement. The
interface should support different attention modes
structurally.
