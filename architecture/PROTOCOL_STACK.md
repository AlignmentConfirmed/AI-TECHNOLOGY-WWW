# Protocol Stack — What Changes and What Stays

## The Current Web Stack

```
APPLICATION    HTTP/HTTPS     (request flat documents)
TRANSPORT      TCP/UDP        (reliable byte delivery)
NETWORK        IP v4/v6       (addressing and routing)
LINK           Ethernet/WiFi  (physical connection)
```

This stack was designed to deliver documents between
computers. It does that well. Every layer below the
application layer works fine for the multi-dimensional
web. We are not replacing the road. We are redesigning
what drives on it.

## What Stays

TCP/IP stays. It moves bytes reliably between machines.
Whether those bytes represent a flat HTML page or a
multi-dimensional information orb, the transport doesn't
care. IPv4 and IPv6 addressing works. DNS works for
human-readable naming. TLS/SSL works for encryption.

The infrastructure is fine. Billions of dollars of
networking equipment doesn't need to be replaced.

## What Changes — The Application Layer

HTTP was designed around a simple model: client requests
a document by URL, server returns the document. The
document is flat text (HTML) with links to other flat
documents. Search is keyword matching against an index
of flat documents.

The multi-dimensional web needs a different application
model:

### 1. Dimensional Object Protocol (DOP)

Instead of requesting a flat document, the client
requests a dimensional object — an information orb with:

- Content (the information itself)
- Domain classification (VERIFIED, BRIDGE, FRONTIER,
  CREATIVE, ARCHIVE)
- Verification level (proven, observed, speculative,
  AI-generated)
- Dimensional metadata (where this object sits relative
  to other objects in information space)
- Provenance (where it came from, how it was verified,
  who created it)
- Context (enough information for the object to be
  understood without seeing the full system it came from)

A DOP response is not a page. It is a self-contained
information bubble that carries its own context.

### 2. Resonance Query Protocol (RQP)

Instead of keyword search, the client sends a measurement
or concept and asks: what in the network converges with
this?

```
CURRENT (HTTP + search engine):
  GET /search?q=gravity+acceleration+measurement
  → returns pages ranked by popularity

PROPOSED (RQP):
  RESONATE {
    domain: VERIFIED,
    concept: [measurement data as Delta],
    threshold: 0.85
  }
  → returns dimensional objects ranked by convergence
    strength with the query, filtered by domain
```

Resonance queries don't match keywords. They match
STRUCTURE. Two measurements that describe the same
phenomenon converge even if they use completely
different vocabulary. This is language-independent
search.

### 3. Verification Receipt Protocol (VRP)

Every exchange of dimensional objects produces an
immutable receipt:

- What was shared
- When it was shared
- The verification level at time of sharing
- Cryptographic hash (tamper-evident)
- Parent receipt (chain continuity)

This creates an auditable trail of how information
moved through the network. If a piece of AI-generated
speculation is later cited as verified fact, the
receipt chain shows WHERE the domain classification
changed and WHETHER it went through proper verification.

### 4. Bubble Exchange Protocol (BEP)

When a user shares information, they don't post a
flat text update. They detach an information bubble
from their private space and hand it to a recipient.

The bubble carries:
- The content at multiple depth levels (summary,
  detail, full)
- Domain classification
- Verification level
- Context sufficient for standalone understanding
- Expiration or permanence flag
- Sharing permissions (can the recipient re-share?)

The recipient holds the bubble. They see the summary.
If they want depth, they zoom in. The bubble is
self-contained. It doesn't require access to the
sender's full information space.

## Migration

The new protocols don't replace HTTP. They extend it.
A dimensional web server can serve both:

- Traditional HTTP requests (backward compatible)
- DOP/RQP/VRP/BEP requests (new functionality)

This means the migration is gradual. Existing websites
continue to work. New dimensional interfaces are added
alongside them. Users can access the same information
through either interface. The dimensional version adds
structure, verification, and spatial organization that
